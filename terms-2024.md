## Rewards
The total reward pool for the DappyKit Hackathon 2024 is up to 20,000 OP tokens. Rewards will be allocated as follows:

### Developer Prizes
- First Place: 5250 OP for the most popular application.
- Second Place: 3150 OP for the second most popular application.
- Third Place: 2100 OP for the third most popular application.
- Participation Reward: The top 100 participants who submit the highest number of eligible applications will each receive 35 OP.

### Promoter Prizes
- First Place: 1250 OP for attracting the highest number of eligible developers.
- Second Place: 750 OP for attracting the second highest number of eligible developers.
- Third Place: 500 OP for attracting the third highest number of eligible developers.
- Participation Reward: An additional 100 promoters who attract the highest number of developers who publish eligible applications will each receive 35 OP.

## Eligibility Criteria for Applications
To be considered an eligible application, the following criteria must be met:

1. The source code of the application must be published on GitHub.
2. The application must integrate and use the DappyKit SDK library - https://github.com/DappyKit/sdk.
3. A demo of the application must be recorded and published on any video service or IPFS, with a maximum length of one minute. The URL to the video must be specified in the GitHub repository.
4. The application must be deployed to one of the following platforms: Farcaster Frames, Telegram, Apple App Store, or Google Play.
5. The link to the production application must be specified in the GitHub repository.
6. If the application competes for the top three places, it is necessary to integrate a publicly verifiable metric for application visits. For example, ClickCaster - https://clickcaster.xyz/.
7. Your application can be either new or previously published. If it was previously published, the integration must be carried out during the hackathon period.

## Participation Rules
1. Developers are allowed to submit more than one application; however, only the best-performing application will be considered for the final evaluation.
2. Only one application per developer can win a prize.
3. Participants may participate in two categories but cannot invite their own accounts or create fake accounts.
4. Any mass or fake accounts created to influence the results will be disqualified, and those involved will be excluded from participating in the hackathon.
